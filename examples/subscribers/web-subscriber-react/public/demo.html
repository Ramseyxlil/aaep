AAEP Web Subscriber Demo

Reference React subscriber for the Agent Accessibility Event Protocol. Demonstrates browser-based AAEP consumption with ARIA live regions.

What this demo shows

When you mount <AAEPSubscriber> in a React app pointing at any AAEP producer, you get:

  • Automatic screen-reader announcements for all 12 AAEP event types
  • Keyboard-accessible confirmation flow with A/R shortcuts
  • Numbered clarification options
  • Automatic reconnection with exponential backoff
  • Optional visible event log for debugging

Run this demo locally

  1. Start any AAEP producer (e.g., from examples/producers/python-minimal/): 
    python -m aaep_minimal_producer.server --port 8080
  2. Install and build this package: 
    cd examples/subscribers/web-subscriber-react
npm install
npm run dev
  3. Open the dev server URL (typically http://localhost:5173).
  4. Turn on a screen reader (NVDA on Windows, VoiceOver on macOS, Orca on Linux) to hear the announcements.
  5. Send a request to the producer to drive a session: 
    curl -X POST http://localhost:8080/sessions \
    -H "Content-Type: application/json" \
    -d '{"user_message": "Send email to alice@example.com"}'

Basic integration

import { AAEPSubscriber } from "@aaep/web-subscriber-react";
import "@aaep/web-subscriber-react/styles.css";

function MyApp() {
  return (
    <div>
      <h1>My AI Assistant</h1>
      <AAEPSubscriber
        endpoint="http://localhost:8080"
        preferredLanguages={["en"]}
        showEventLog={true}
      />
    </div>
  );
}

Multilingual

Configure preferred languages to support Yoruba, Hausa, Igbo, or any other language a producer emits: 

<AAEPSubscriber
  endpoint="http://localhost:8080"
  preferredLanguages={["yo", "en"]}  // Prefer Yoruba, fall back to English
/>
